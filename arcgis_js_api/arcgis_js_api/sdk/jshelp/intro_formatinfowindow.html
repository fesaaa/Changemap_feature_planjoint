Format Info Window Content

The InfoTemplate class is used to define the content and title for an Info Window. If you are using version 2.2 or greater, you can define the content and title using either a string or function. Prior to version 2.2, you can only use a string.

Resize Info Window

By default, the info window size is set to 270 pixels wide and 400 pixels high. If the content you want to display is larger than this size scroll bars are displayed to allow users to view the content. In some cases you may want to resize the info window to be larger or smaller than the default so your content displays well in the info window. To resize the info window use the resize method to specify the new width and height. 

  infoWindow.resize(300,200);

Format content using a string

To format content using a string, you will create a string value that defines the content to display. The string can include HTML markup, attribute placeholders and formatting functions. Let's look at a few examples of ways you can format content using a string:

  • Simple String

    In this example, a simple string is used to display text in an info window. 

      myTemplate.content = "This is a location of interest";

  • HTML Markup

    You can also include HTML markup in the string, in this example the info window will contain several paragraphs with some content in bold. 

      myTemplate.content = "This is some content with
    <b>HTML</b> markup!<p>This is a paragraph
    </p><p>Another Paragraph</p>"

  • Placeholders

    Use placeholders, created using ${}, to define the attribute value that should display in the info window. At run-time parameter substitution occurs and the placeholder is replaced with the actual attribute value for the selected graphic. In this example, OIL_CUMULATIVE and YEAR are valid attribute fields and will be replaced with the value when the info window is displayed. 

      myTemplate.content = "<b>Status</b>: ${STATUS}" +
    "<br><b>Cummulative Gas</b>: ${CUMM_GAS} MCF" +
    "<br><b>Total Acres</b>: ${APPROXACRE}" +
    "<br><b>Avg. Field Depth</b>: ${AVG_DEPTH} meters";

  • Formatting Functions

    At 2.2,formatting support for placeholders was added. You can define a custom formatting function or format date and time values using one of the out of the box formatters. The syntax for setting formatting options is: 

      ${FIELD_NAME:FORMAT_FUNCTION_NAME(OPTION_A:,OPTION_B:,...)}
    View the table below for a list of available formatter functions:
    Formatter Function Options
    DateString Formats the date into a human readable format. Supports the following options:
    • hideTime: Hide the time component of the date string. The default value is false.
    • local: Display time in the local time zone. The default value is false. When false the date will display in UTC.
    • systemLocale: Display time in the system locale (local time zone). The default value is false.
    Pattern Example
    ${EVENT_DATE:DateString} Thu, 15 May 1986 04:26:32 GMT
    ${EVENT_DATE:DateString(local: true, hideTime: true)} Fri May 07 2004
    DateFormat Converts date fields into a human readable format using the dojo/data/locale.format method. Supports the same set of options as dojo/date/locale.format.
    Pattern Example
    ${EVENT_DATE:DateFormat} 3/8/08 4:00 PM
    ${EVENT_DATE:DateFormat(selector: 'date', fullYear: true)} 4/8/2009
    ${EVENT_DATE:DateFormat(datePattern: 'h \\'o\\'\\'clock\\' a, zzzz', selector: 'date')} 4 o'clock PM, Pacific Standard Time
    NumberFormat Formats numbers into various formats using the dojo/number.format method. Supports the same set of options as dojo/number.format.
    Pattern Example
    ${Magnitude:NumberFormat} 4.237854
    ${Magnitude:NumberFormat(places:2)} 4.2
    Custom Function Define a function that will format the attribute value. 
    ${DIFF:compareValues}

function compareValues(value, key, data){
  var diff = data.POP2007 - data.POP2000;
  var result = diff > 0 ? "up.png" : "down.png";
  var pctChange = (diff * 100) / data.POP2000;
  return "<img src='" + result + "'/>" + 
  "  <span style='color: " + (pctChange < 0 ? "red" : "green") + ";'>" 
  + dojo.number.format(pctChange, { places: 3 }) + "%</span>;
}

Format content using a function

Sometimes you want to display more than just text in an info window. For example, you might want to display a chart, categorize information into tabs or use a Dojo widget (i.e. Dijit) in the info window. In such cases, you can write a function that returns either a string, a reference to an HTML element, or a deferred object. When the graphic is clicked, the function is executed and the return value displays in the info window. Note that the function is passed a reference to the current graphic as an argument, which provides access to the graphic's attribute information.

  • String

    Return a simple string, note that it cannot contain placeholders. In this example a link to a wikipedia article is constructed from the attribute values. 

    function getTextContent(graphic) {
  var commName = graphic.attributes.qSpecies
  var hlink = names[0].split(" ");
  var sciName = hlink[0] + "_" + hlink[1];
  return "<b>" + commName +
    "</b><br /><a target='_blank' href=http://en.wikipedia.org/wiki/"
    + sciName  +">Wikipedia Entry</a>";
}

  • Reference to an HTML element

    Return a reference to an HTML element by programatically creating an HTML element using either dojo.create or create a Dojo dijit and return the dijit's dom node.

    Dojo.create is a convenience method that encapsulates the document.createElement and Node.appendChild methods. 

    // legacy
var myElement = dojo.create("div",{
  innerHTML: "Text Element inside an HTML div element."
});

// AMD
require(["dojo/dom-construct"], function(domConstruct){
  var node = domConstruct.create("div", { innerHTML: "Text Element inside an HTML div element." });
});

    Create a Dojo dijit and return the dijit's dom node. 

    var titlePane = new dijit.TitlePane{
  title: "Details",
  content: "Welcome to Paris, capital of France, located on the river Seine."
});
return titlePane.domNode;

  • Deferred Object

    In some cases, the content you want to display is not readily available; for example, it may be content that you need to download from the server using esri.request. The esri.request method returns an instance of Dojo.Deferred that represents the asynchronous network request. When the user clicks on a graphic, the code that executes this function will see that it has returned a deferred object, and will wait until the deferred has finished to display the value.

    This code sample executes when the user clicks the polyline on the map. This polyline is provided as input to a Server Object Extension that generates an elevation profile. Once the profile is generated it will display in the info window. 

    function getTextContent(graphic) {
  var deferred = new dojo.Deferred();
  params.InputPolyline = dojo.toJson(graphic.geometry.toJson());
  
  esri.request({
    url: soeURL,
    content: params,
    callbackParamName: "callback",
    load: function(fset) {
      deferred.callback("<img src='" + fset.profileImageUrl + "'/>");
    },
    error: function(error) {
      deferred.errback("Error occurred while generating profile");
    }
  });
  return deferred; 
}
